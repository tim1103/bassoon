# 系统架构设计文档

## 1. 系统概述

浙江省新高考选课排课系统是一个基于 Web 的独立运行系统，用于支持新高考改革下的选课和排课需求。

### 1.1 核心特性

- **适用政策**: 浙江省新高考改革（7 选 3 模式）
- **系统定位**: 独立运行的选课排课系统，不依赖外部数据中心
- **核心业务**: 行政班 + 走班制混合排课、语数英分层教学、跨年级排课
- **数据管理**: 系统内独立维护基础数据，支持批量导入导出

## 3. 模块设计

### 3.1 数据层（Data Layer）

**职责**: 基础数据的持久化和管理

**核心类**:
- `DataManager`: 数据管理器，负责 CRUD 操作
- `ExcelHandler`: Excel 处理器，负责导入导出

**数据实体**:
| 实体 | 文件名 | 主要字段 |
|------|--------|----------|
| 校区 | campuses.json | id, name, address |
| 班级 | classes.json | id, name, grade, type |
| 教师 | teachers.json | id, name, gender, subjects |
| 学生 | students.json | id, student_id, name, admin_class |
| 教室 | classrooms.json | id, code, name, capacity |
| 课程 | courses.json | id, name, teacher_id, periods |

### 3.2 选课层（Selection Layer）

**职责**: 选课方案制定和学生选课管理

**核心服务**: `SelectionService`

**功能模块**:
1. **规则引擎**
   - 互斥规则（某些科目不能同时选）
   - 必选规则（选了 A 必须选 B）
   - 禁选规则（特定组合禁止）
   - 人数限制（最大/最小班级人数）
   - 性别限制

2. **选课流程**
   ```
   方案制定 → 学生预选 → 结果确认 → 人工调整
   ```

3. **分班优化**
   - 成绩均衡算法
   - 男女比例均衡
   - 原行政班集中

**选科组合**: 7 选 3，共 C(7,3) = 35 种组合

### 3.3 排课层（Scheduling Layer）

**职责**: 自动生成最优课表

**核心服务**: `SchedulingService`

**排课算法**: 遗传算法（Genetic Algorithm）

```
初始化种群 → 计算适应度 → 选择 → 交叉 → 变异 → 迭代优化
```

**关键组件**:

1. **编码方案**
   - 染色体表示：时间片 + 教室 + 教师的三元组
   - 基因位置：每个教学任务

2. **适应度函数**
   ```python
   fitness = 
     - 教师冲突 * 100
     - 教室冲突 * 100
     - 未排入任务 * 50
     + 课时分布均匀度 * 10
     + 教师空闲优化 * 5
   ```

3. **约束条件**
   - 硬约束：教师不能同时上两节课、教室容量限制
   - 软约束：课时分布均匀、避免教师长时间空闲

4. **遗传算子**
   - 选择：锦标赛选择法
   - 交叉：单点交叉
   - 变异：随机重新安排时间

**排课模式支持**:
- 行政班排课
- 教学班排课
- 先选后排
- 先排后选
- 大走班
- 定二走一

### 3.4 应用层（Application Layer）

**职责**: 用户界面和业务应用

**功能模块**:
1. **查询统计**
   - 数据概览统计
   - 选课统计分析
   - 课表查询

2. **数据导出**
   - 学生课表
   - 教师课表
   - 班级课表
   - 教室课表

3. **报表打印**
   - 各类课表打印
   - 统计报表导出

## 4. 数据流设计

### 4.1 选课流程数据流

```
用户创建方案 → 规则配置 → 保存方案
                       ↓
学生提交志愿 → 规则校验 → 保存结果
                       ↓
                   分班优化 → 生成教学班
```

### 4.2 排课流程数据流

```
读取基础数据 → 生成教学任务 → 遗传算法排课
                                    ↓
                              适应度评估
                                    ↓
                              迭代优化
                                    ↓
                              生成多套方案
                                    ↓
                              输出最终课表
```

## 5. API 设计

### RESTful API 规范

```
GET    /api/{resource}      - 获取资源列表
POST   /api/{resource}      - 创建资源
PUT    /api/{resource}/{id} - 更新资源
DELETE /api/{resource}/{id} - 删除资源
```

### 核心 API

| 端点 | 方法 | 描述 |
|------|------|------|
| /api/campuses | GET/POST | 校区管理 |
| /api/teachers | GET/POST | 教师管理 |
| /api/students | GET/POST | 学生管理 |
| /api/classes | GET/POST | 班级管理 |
| /api/classrooms | GET/POST | 教室管理 |
| /api/import/{type} | POST | 导入 Excel |
| /api/export/{type} | GET | 导出 Excel |
| /api/selection/scheme | POST | 创建选课方案 |
| /api/selection/submit | POST | 提交选课 |
| /api/selection/optimize | POST | 分班优化 |
| /api/scheduling/generate | POST | 生成课表 |
| /api/scheduling/timetable | GET | 获取课表 |
| /api/stats/overview | GET | 统计概览 |

## 8. 扩展性设计

### 8.1 数据库扩展
当前使用 JSON 文件存储，可通过修改 `DataManager` 轻松切换到：
- SQLite（单机版）
- MySQL（多用户版）
- PostgreSQL（企业版）

### 8.2 算法扩展
排课算法采用策略模式，可替换为：
- 模拟退火算法
- 约束满足问题（CSP）求解器
- 禁忌搜索算法
- 蚁群算法

### 8.3 功能扩展
预留接口支持：
- 多校区联动
- 跨年级排课
- 走班制优化
- 成绩分析
